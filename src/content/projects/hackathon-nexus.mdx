I've volunteered at several hackathons at Gonzaga University and Eastern Washington University, and there's a common issue with all of them.

**Everything is pen and paper.** As a mentor or judge, I cannot:

- Easily see project code for each team in a repo
- Know which team might need a mentor to answer questions or provide guidance
- Coordinate votes for judging
- Gain insight into teams progress throughout the day (it'd be fun to have an "afraid of commit-ment" award for the least amount of commits to a repo)

**I aim to fix this by creating an all in one hackathon nexus.**

### Features:
- Registration
- Team management
- Git repo links
- "Request a Mentor" queue so volunteers know where to go
- Judging in line
